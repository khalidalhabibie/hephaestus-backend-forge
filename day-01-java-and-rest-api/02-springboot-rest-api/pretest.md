# Pretest - Spring Boot REST API

Jawab pertanyaan berikut dengan bahasa sendiri sebelum membaca materi Spring Boot REST API.

### 1. Apa itu Spring Boot?

Jawaban:

```text
Framework yang digunakan untuk mempermudah pembuatan aplikasi terutama dalam pembuatan backend
```

### 2. Apa itu REST API?

Jawaban:

```text
Rest API merupakan API yang menerapkan prosedur API dalam implementasinya 
```

### 3. Apa itu HTTP?

Jawaban:

```text
HTTP merupakan protokol yang digunakan dalam mengirim dan juga menerima data
```

### 4. Apa itu JSON?

Jawaban:

```text
JSON merupakan bentuk format data yang didalamnya biasa terdapat kumpulan value yang memiliki key dan value.
```

### 5. Apa itu endpoint?

Jawaban:

```text
Enpoint merupakan tujuan atau alamat dari data akan dikirimkan  
```

### 6. Apa itu request?

Jawaban:

```text
request merupakan prosedur atau langkah dalam melakukan permintaan data menggunakan API 
```

### 7. Apa itu response?

Jawaban:

```text
Response merupakan jawaban atas permintaan data melalui API termasuk pesan apakah data berhasil dikirimkan atau tidak
```

### 8. Apa fungsi GET?

Jawaban:

```text
Fungsi get merupakan method dalam API yang digunakan untuk meminta atau melihat data melalui API
```

### 9. Apa fungsi POST?

Jawaban:

```text
Fungsi POST merupakan method yang digunakan untuk menambahkan data melalui API
```

### 10. Apa fungsi PUT?

Jawaban:

```text
Fungsi POST merupakan method yang digunakan untuk memperbarui data melalui API
```

### 11. Apa fungsi DELETE?

Jawaban:

```text
Fungsi POST merupakan method yang digunakan untuk menghapus data melalui API
```

### 12. Kapan menggunakan 200 OK?

Jawaban:

```text
Menggunakan 200 OK ketika API berhasil atas request GET atau berhasil mengirim data yang diminta
```

### 13. Kapan menggunakan 201 Created?

Jawaban:

```text
201 created ini digunakan ketika request untuk melakukan POST atau penambahan data berhasil dilakukan melalui API
```

### 14. Kapan menggunakan 400 Bad Request?

Jawaban:

```text
Menggunakan 400 bad request ketika data gagal diminta atau ditambahkan akibat ada keasalah proses didalamnya atau tidak berhasil menjalankan request dengan baik
```

### 15. Kapan menggunakan 404 Not Found?

Jawaban:

```text
404 Not found ini ketika data yang diminta tidak ada ataupun ketika menambahkan data tabelnya tidak ada
```

### 16. Kapan menggunakan 500 Internal Server Error?

Jawaban:

```text
500 internal server error ini digunakan ketika servernya error atau tidak berjalan dengan baik
```

### 17. Apa itu path parameter?

Jawaban:

```text
Path parameter merupakan lokasi secara lebih spesifik dari data yang dituju
```

### 18. Apa itu query parameter?

Jawaban:

```text
Query paramater merupakan filter yang ditambahkan untuk memberi spesifikasi data yang dituju
```

### 19. Apa itu request body?

Jawaban:

```text
Request body ini merupakan cara kita meminta data melalui proses request
```

### 20. Apa tugas Controller?

Jawaban:

```text
Controller sebagai penghubung untuk menerima dan mengirim data
```

### 21. Apa tugas Service?

Jawaban:

```text
Tugas service adalah menyimpan logic yang mengelola suatu program
```

### 22. Apa itu DTO?

Jawaban:

```text
DTO adalah singkatan dari Data Transfer Object, yaitu sebuah objek (class) yang digunakan untuk mengirim data antar layer dalam aplikasi
```

### 23. Apa itu Model?

Jawaban:

```text
Model merupakan bagian dari program untuk blueprint atas atribut yang ada di suatu program
```

### 24. Kenapa business logic tidak ditaruh di Controller?

Jawaban:

```text
Business logic tidak ditaruh di controller untuk mempermudah proses debug atas error dalam bussines logic atau terkait koneksi ke database
```

### 25. Kenapa request/response perlu DTO?

Jawaban:

```text
Request/response perlu DTO karena kita ingin mengontrol bagaimana data masuk dan keluar dari aplikasi, bukan asal lempar data dari Entity (database)
```
