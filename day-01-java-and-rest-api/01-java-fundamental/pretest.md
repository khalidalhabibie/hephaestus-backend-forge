# Pretest - Java Fundamental

## Objective

Pretest ini bertujuan mengukur pemahaman awal tentang Java Fundamental sebelum membaca materi dan mengerjakan exercise.

## Instructions

- Jawab dengan bahasa sendiri.
- Tidak perlu takut salah.

- Jika belum tahu, tulis apa yang kamu pahami saat ini.

Estimasi waktu: 20-30 menit.

## Section A - Basic Syntax

### 1. Apa itu variable?

Jawaban:
variabel seperti wadah yang dapat diisi dengan nilai/value.

### 2. Apa itu data type?

Jawaban:
Data type itu untuk menentukan data yang akan kita gunakan seperti apa, contoh jika kita mau masukin data tentang uang atau nominal kita bisa gunakan type data float atau double.

### 3. Sebutkan contoh tipe data untuk text, number, dan true/false.

Jawaban:
text : String
number : float, double, integer
true/false : boolean

### 4. Apa perbedaan variable declaration dan assignment?

Jawaban:
variable declaration: mendeklarasikan variabel tanpa memberikan value
contoh: String nama

variable assignment: mendeklarasikan variabel dan sekalian memberikan value
contoh: nama = fadhly

### 5. Apa itu camelCase?

Jawaban:
camelcase itu gaya penulisan dari kecil ke besar 
contoh: namaMobil

## Section B - Class and Object

### 6. Apa itu class?

Jawaban:
blueprint atau template untuk membuat object

### 7. Apa itu object?

Jawaban:
object itu hasil dari class

### 8. Apa perbedaan class dan object?

Jawaban:
class itu seperti desain yang akan kita gunakan untuk objek sedangkan objek itu hasil dari class yang dapat digunakan (asli/nyata dan bisa digunakan)

## Section C - Field, Method, Constructor

### 9. Apa itu field?

Jawaban:
variabel yang berada dalam class

### 10. Apa itu method?

Jawaban:
method itu logika yang akan kita gunakan, class dan objek yang sudah kita deklarasikan akan kita implementasikan dengan menggunakna method 

### 11. Apa itu parameter?

Jawaban:
variabel yang kita gunakan dalam method/function tersebut. yaitu variabel yang sudah kita deklarasikan sebelumnya di dalam class

### 12. Apa itu return value?

Jawaban:
return value itu untuk menyimpan dan mengembalikan nilai kepada method dan jika dipanggil itu dapat menampilkan nilai tersebut

### 13. Apa itu constructor?

Jawaban:
method yang berada di dalam class untuk menginisialisasi object 

## Section D - Access Modifier

### 14. Apa fungsi private?

Jawaban:
agar tidak dapat digunakan di class lain, jika ingin menggunakan bisa dengan setter getter

### 15. Apa fungsi public?

Jawaban:
agar bisa digunakan disemua class, jadi lebih mudah diakses dan lebih flexible

### 16. Kenapa field biasanya dibuat private?

Jawaban:
agar tidak bisa diotak atik dengan class lain, jadi jika ada yang ingin menggunakan bisa dengan stter dan getter

## Section E - Getter and Setter

### 17. Apa fungsi getter?

Jawaban:
untuk memanggil variabel yang bersifat private, dan hanya bisa menampilkan data saja

### 18. Apa fungsi setter?

Jawaban:
untuk memanggil variabel yang bersifat private, dan selain bisa menampilkan itu juga bisa merubah data tersebut

### 19. Kenapa tidak semua field dibuat public?

Jawaban:
untuk kemamanan, agar jika ada class lain yang ingin menggunakan bisa lebih berhati hati karena berupa private dimana biasanya jika berbentuk private itu variabel yang penting

## Section F - Collection

### 20. Apa itu List?

Jawaban:
kumpulan data yang berurutan dengan data yang sama
contoh list nama pasti isinya nama semua ga ada yang umur

### 21. Apa itu Map?

Jawaban:
kumpulan data yang memiliki key value, datanya bisa beda beda

### 22. Kapan menggunakan List dan kapan menggunakan Map?

Jawaban:
jika ingin mencari sesuai urutan dan data yang sama semua bisa menggunakan list
jika ingin mencari sesuai dkey dan data yang beda beda bisa menggunakan map

## Section G - OOP Basic

### 23. Apa itu encapsulation?

Jawaban:
pemberian akses modifier kepada class lain seperti public, private, protected, abstract

### 24. Apa itu interface?

Jawaban:
wajib menggunakan dari variabel dan function parents, tidak bisa menambahkan variabel ataupun function

### 25. Apa itu abstract class?

Jawaban:
hybrid, bisa menggunakan variabel dan function parents semuanya dan bisa juga tidak menggunakan variabel dan function parents semuanya 

### 26. Apa perbedaan interface dan abstract class?

Jawaban:
abstract harus menggunakan override (yang ingin digunakan) tapi kalo interface udah pasti kepake semuanya
